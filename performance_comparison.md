# Сравнение производительности операций

**Тестирование на 10000 записей**

| Операция | Тип данных | Кол-во записей | Время (мс) | Алгоритм/Подход | Примечания |
|----------|------------|----------------|------------|------------------|------------|
| INSERT | analFun | 10000 | 19498 | Batch Save | Базовая вставка |
| INSERT | compFun | 10000 | 18274 | Batch Save | Базовая вставка |
| INSERT | Сравнение | 10000 | - | - | compFun быстрее на 1224мс |
| SELECT | analFun | 1 | 93 | FindByName | Одиночный поиск |
| SELECT | analFun | 10000 | 103 | FindAll | Полная выборка |
| SELECT | analFun | 1213 | 86 | Stream Filter | Фильтрация в памяти |
| SORT | analFun | 10000 | 6 | Stream Sorted | Сортировка в памяти |
| SORT | analFun | 10000 | 9 | Stream Sorted | Сортировка по типу |
| SORT | analFun | 10000 | 3 | Collections.sort | Ручная сортировка |
| DELETE | analFun | 1 | 22 | Delete One | Удаление одной записи |
| DELETE | analFun | все | 17519 | Delete All | Массовое удаление |

## Выводы:
- Batch операции значительно быстрее одиночных
- Сортировка в памяти эффективна для средних объемов данных
- Поиск по индексируемым полям работает быстрее
